# engine/botlib/l_log.h

## File Purpose
- 	This module implements a simple logging subsystem for the bot library, providing file I/O, timestamped writes, and lifecycle management.

## Core Responsibilities
- 	- Manage the lifetime of the log file used by botlib functions;
- 	- Provide formatted logging routines with optional timestamping;
- 	- Centralize error handling for missing or failed log operations;
- 	- Expose the underlying FILE* for legacy or lower‑level use;
- 	- Ensure that log output is flushed immediately to disk.

## Key Types and Functions
- Log_Open(char* filename) — opens a log file when logging is enabled;
- Log_AlwaysOpen(char* filename) — opens a log file for writing if none is open and initializes global state;
- Log_Close() — closes the current log file if open;
- Log_Shutdown() — shuts down logging by calling Log_Close;
- Log_Write(char* fmt, ...) — writes a formatted message to the log file without a newline;
- Log_WriteTimeStamped(char* fmt, ...) — writes a timestamped, counter‑prefixed entry followed by CRLF;
- Log_FilePointer() — returns the FILE* currently open for logging;
- Log_Flush() — forces the logfile buffer to be written to disk.

## Important Control Flow
- Log_Open checks a config variable and delegates to Log_AlwaysOpen if logging is enabled; Log_AlwaysOpen opens the file, populates global logfile structure, and reports success or failure; Log_Close and Log_Shutdown close an open logfile and reset the global handle; Log_Write outputs formatted strings to the logfile using vfprintf and flushes the stream; Log_WriteTimeStamped prefixes each entry with a counter and timestamp derived from botlibglobals.time, then writes and flushes; Log_FilePointer returns the raw FILE* for external use; Log_Flush calls fflush on the current logfile pointer if valid.

## External Dependencies
- 	- Standard C library <stdio.h> for FILE, vfprintf, fflush;
- 	- Bot library globals for time tracking (botlibglobals.time);
- 	- Config variable handling (e.g., CVar value 'log');
- 	- Error printing via botimport.Print;
- 	- Constants such as MAX_LOGFILENAMESIZE.

## How It Fits
- 	Acts as the I/O backend for diagnostic output generated by bot logic;
- 	Keeps bot logs separate from console or engine logs;
- 	Relies on global botlib state for time and configuration;
- 	Provides a minimal, portable logging API that avoids dependencies on external logging frameworks.
